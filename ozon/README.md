# Ozon 商品 AI 半自动生成助手

这是一个本地 Streamlit 工具，用来把宠物服装图片识别成 Ozon 商品资料草稿，并写入你提供的 Ozon 中文 Excel 模板。工具也保留了原来的 Ozon Seller API 上传页面，但默认 dry-run，不会自动上架。

## 现在 MVP 能做什么

- 扫描一个商品图片文件夹。
- 调用 OpenAI API 识别商品内容、主图、详情图、尺码图。
- 生成俄文标题、简介、卖点、标签。
- 先保存 AI 识别 JSON，再进入人工审核。
- 人工修改确认后，写入 Ozon 中文 Excel 模板。
- 把图片复制整理到 `output_images/商品货号/`。
- 输出处理报告到 `data/output/处理报告.json`。
- 如果没有 OpenAI API Key 或额度，可以用“不用 OpenAI，手动建商品草稿”检查 Excel 写入流程。

## 安装

在项目目录运行：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

如果你已经用旁边的 `Ozon商品助手\.venv` 安装过依赖，也可以直接用那个 Python 启动。

## 配置 .env

复制 `.env.example` 为 `.env`，填写：

```env
OPENAI_API_KEY=你的 OpenAI API Key
OPENAI_MODEL=gpt-4.1-mini
CUSTOM_GPT_URL=
CUSTOM_GPT_API_KEY=

OZON_CLIENT_ID=
OZON_API_KEY=
OZON_BASE_URL=https://api-seller.ozon.ru
OZON_MOCK_MODE=true
```

`OPENAI_MODEL` 可以换成你账号支持的视觉模型。Ozon 上传页面需要 Ozon Seller API 密钥；只生成 Excel 时不需要填写 Ozon 密钥。

如果你没有 OpenAI 额度，但有自己的 GPT/识别程序，可以把接口地址填到 `CUSTOM_GPT_URL`。程序会优先调用自定义 GPT，发送 `prompt`、`images` 和 `schema`，接口返回 JSON 对象即可，也可以返回 `{ "result": {...} }`、`{ "analysis": {...} }` 或 `{ "text": "{...}" }`。

如果你只有 ChatGPT Plus 网页账号，可以使用页面里的“GPT Plus 手动识别”：点击“复制 GPT 提示词”，打开 ChatGPT，把商品图片和提示词一起发给 ChatGPT，再把 ChatGPT 返回的 JSON 粘贴回软件。

## 启动

```powershell
.\.venv\Scripts\python.exe -m streamlit run app.py
```

如果当前项目 `.venv` 没装好，可以用：

```powershell
..\Ozon商品助手\.venv\Scripts\python.exe -m streamlit run app.py
```

## 图片放在哪里

推荐先用单个商品模式：

```text
ozon_auto_listing/
  input_images/
    商品主图.png
    详情图.png
    尺码图.png
```

也可以在页面里填写其他图片目录，例如 `data/images`。

## 使用流程

1. 打开页面后进入“AI 生成 Excel 草稿”。
2. 上传或填写 Ozon 中文模板路径。
3. 填写商品图片文件夹。
4. 填写默认货号、价格、重量、包装尺寸。
5. 填写商品名称（中文），程序会按常用宠物商品词自动生成俄文名称，也可以手动修改。
6. 填写服装类型；如果不确定可以留空，不会自动乱猜。
7. 如果已有公网图片链接，填写主图/附图链接；没有链接时 Excel 图片列会留空，后续手动上传图片。
8. 如果有 OpenAI 额度或 `CUSTOM_GPT_URL`，点击“开始识别并生成商品资料”。
9. 如果只有 ChatGPT Plus 网页账号，用“GPT Plus 手动识别”：点击“复制 GPT 提示词”，把提示词和图片发到 ChatGPT，再把返回 JSON 粘贴回软件并导入。
10. 如果没有识别接口，先填写颜色/尺码，再点击“不用 OpenAI，手动建商品草稿”。
11. 在人工审核区修改俄文标题、简介、标签、价格、重量等。
12. 点击“确认这些资料”。
13. 点击“生成已填写 Excel”。

### 多颜色和多尺码

人工审核区可以填写“颜色/款式列表”和“尺码明细”。生成 Excel 时会按 `颜色/款式 × 尺码` 展开为多行：1 个颜色 5 个尺码生成 5 行，2 个颜色 5 个尺码生成 10 行。

颜色/款式每行一个：

```text
红色|красный
蓝色|синий
```

常见颜色只填中文也可以，颜色名称默认会按主色填写，例如主色是 `绿色`，颜色名称也写 `绿色`。

页面里还有“手动主色（中文，优先使用）”。如果 GPT 把颜色识别成“绿色恐龙”之类，手动填 `绿色`，Excel 会按主色写 `绿色`。货号里的颜色也会用中文拼音码，例如黑色 `HEI`、绿色 `LV`。

尺码每行一个。可以只填尺码，也可以按 `尺码|胸围cm|背长cm|颈围cm|价格|重量g|长mm|宽mm|高mm` 覆盖默认数据。胸围和颈围可以填范围，例如 `32-38`，程序会写入 Ozon 模板里的最小/最大胸围、最小/最大颈围；如果只填一个数，只写入最大胸围或最大颈围。背长会按原值写入“背部长度，厘米”。
价格、重量、包装尺寸只有明确填写在后面的字段时才写入；没有提供就留空，不会用胸围或背长去猜。
`2XL` 会自动改成 `XXL`。

```text
S|32-38|25|20-26|99|120|280|200|25
M|38-44|30|24-30|109|140|300|220|30
L|44-50|35|28-34|119|160|320|240|35
```

输出位置：

```text
output_images/商品货号/
output_excel/Ozon_AI_已填写_日期时间.xlsx
data/output/product_json/
data/output/处理报告.json
```

## Excel 模板写入说明

程序会按表头名称匹配，不写死列号。优先写入这些字段：

- 货号*
- 商品名称
- 简介
- 价格，CNY*
- 毛重，克*
- 包装宽度，毫米*
- 包装高度，毫米*
- 包装长度，毫米*
- 主图链接*
- 附加图片链接
- 品牌*
- 型号名称（针对合并为一张商品卡片）*
- 动物的性别
- 商品颜色
- 颜色名称
- 类型*
- 专为*
- 服装类型
- #主题标签
- 动物服装/配件的尺寸
- 最小胸围，厘米
- 最大胸围，厘米
- 背部长度，厘米
- 最小颈围，厘米
- 最大颈围，厘米
- 材料
- 品牌国家
- 原产国

如果模板里没有某个字段，程序会跳过，不会报错。

## 重要限制

- 工具不会自动登录 Ozon。
- 工具不会自动上架。
- 工具不会绕过平台风控。
- `description_category_id` 和 `type_id` 不能凭空猜测，需要你从 Ozon 后台或官方文档确认后再用于 API 上传。
- AI 识别结果必须人工核对后再使用。
- 如果图片中没有明确材料、尺寸、重量，不要让 AI 乱填，页面里可以手动留空或修改。

## 常见问题

### 页面提示 Excel 缺少必填列

这是 API 上传页面的校验结果。如果你上传的是空模板，需要先到“AI 生成 Excel 草稿”页面生成已填写 Excel。

### 未检测到 OPENAI_API_KEY

在 `.env` 中填写 `OPENAI_API_KEY`，或者先点击“不用 OpenAI，手动建商品草稿”。

### 图片字段是文件名，不是 URL

生成 Excel 阶段先写本地文件名，方便核对。真实 Ozon API 上传图片通常需要公网可访问 URL，后续可以接图床或对象存储。

### 商品名称模板规则

程序已加入推荐商品名称模板：

```text
商品类型 + 款式/功能 + 适用对象 + 使用场景/核心卖点
```

俄语模板参考：

```text
Тип товара + фасон/функция + для кого + сценарий/ключевое преимущество
```

标题里不要写颜色、尺码、SKU、货号、库存、物流词、促销词。颜色只写到 `color` / `color_name_ru` 字段，尺码只写到 `sizes` 字段。

示例：

```text
Спасательный жилет для собак, регулируемый жилет для плавания и отдыха на воде
Дождевик-комбинезон для собак, водонепроницаемая одежда для прогулок в дождь
Обувь для собак для прогулок, защитные ботинки для лап
```
